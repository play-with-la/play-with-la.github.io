# 玩转线性代数（play with linear algebra）

本项目从以下的一段提示词开始，主要使用 GitHub Copilot 中的 Claude Opus 4.5(Preview) 大模型完成，后期由于该大模型的高级请求快用完了，因此也尝试了 Claude Sonnet 4.5、Claude Haiku 4.5、GPT-5.1-Codex (Preview)、Grok Code Fast 1 的模型。针对本项目生成的代码，总体来说 Claude 系列模型表现最好；用 GPT-5.1-Codex 完成了几个函数功能的补全，表现也还可以；Grok Code 生成的代码没有满足要求。

> 我想创建一个交互式的线性代数学习系统，它使用现代的网页技术和相关的js库，如Bootstrap、math.js、mathjax.js、three.js等来呈现，第三方的js库通过script标签在index.html页面中加载，确保整个项目可以通过双击index.html来运行。整个系统的构思我还不是很清晰，一个大概的样子是：有一个主页面，中间是一个显示区域，有一个二维或三维的网格，主要作用是用来呈现向量或者图形；左边是一个功能区域，有四组卡片组成，第一个卡片是一个可折叠的系统设置区域，在第二个卡片中可以输入一个向量，在第三个卡片中可以输入一个矩阵，第四个卡片则是一个下拉列表，里面有各种功能函数（这些功能函数主要用来演示线性代数中的各种概念，比如矩阵可以作为某种变换，矩阵的特征值、特征向量，等等）。你觉得我描述是否还有不太清楚的地方，请补充一下。

在决定使用 Claude 模型之前，用上面的提示词也测试了 Gemini 3 Pro (Preview) 和 GPT-5.1-Codex (Preview) 的表现，效果都不太让人满意，而 Claude 生成的代码则符合预期，因此选择采用 Claude 的模型。本项目完整的提示词历史（v1.0 版本）可以在[这里](./assets/chat_history.md)查看。



## 功能介绍（大模型生成）

本项目是一个交互式的线性代数可视化学习系统，支持 2D 和 3D 两种显示模式，帮助用户直观理解线性代数中的核心概念。

### 主要功能模块

#### 1. 系统设置
- **显示模式切换**：支持 2D/3D 模式自由切换
- **主题切换**：亮色/暗色主题
- **网格与坐标轴**：可控制网格、坐标轴、刻度标签的显示
- **3D 专属设置**：Z 轴网格、标准坐标轴网格面、子空间网格面
- **动画速度**：可调节动画播放速度

#### 2. 向量管理
- 添加、编辑、删除向量
- 自定义向量颜色和名称
- 拖拽向量端点实时调整向量（2D 模式）
- 双击编辑向量坐标和名称
- 可见性切换

#### 3. 矩阵管理
- 支持 2×2 和 3×3 矩阵输入
- 预置常用变换矩阵（旋转、缩放、剪切、反射、投影等）
- 矩阵运算：加法、减法、标量乘法、矩阵乘法（支持 2-3 个矩阵连乘）
- 矩阵列表管理，悬停查看矩阵内容

#### 4. 图案管理
- **2D 模式**：
  - 预置图形（目前只有圆形）
  - 手动绘制自定义图案
  - 支持调整、拖动图案
- **3D 模式**：
  - 预置立体图形（球体、椭球、正方体、长方体、圆柱、圆锥）
  - 可调节图形参数（半径、尺寸、分段数等）
  - 实时编辑已添加的图形

#### 5. 空间管理（子空间可视化）
- 选择基向量创建子空间
- 支持 1D（直线）、2D（平面）、3D（全空间）子空间
- 子空间网格和基向量轴线可视化

#### 6. 功能演示
- **向量运算**：
  - 向量加法/减法（带动画演示）
  - 标量乘法
  - 向量单位化
  - 向量投影（带动画演示）
  - 叉积（3D 模式）
- **矩阵变换**：
  - 映射向量：应用矩阵变换到向量
  - 映射图案：应用矩阵变换到图案
  - 动力学轨迹：可视化矩阵迭代作用的效果
  - 方程组解集：演示线性方程组的解集
  - 二次型：演示二次曲线和二次曲面

### 技术栈

- **前端框架**：Bootstrap 5.3
- **3D 渲染**：Three.js
- **数学计算**：Math.js
- **数学公式渲染**：MathJax
- **图标**：Bootstrap Icons

### 使用方式

克隆本项目，并在该项目的根目录（`index.html` 文件所在的目录）下开启一个 http 服务（可以在命令行下执行 `python -m http.server`，或通过 VS Code 中的 `Live Server` 插件）即可在浏览器中演示。
